Processing

In CSS, the page is processed from top to bottom. That means that CSS on the bottom of the page overrides CSS at the bottom of the page.

Selectors

When CSS elements are laid out in a CSS stylesheet with only spaces between them, the CSS rule applies to nested elements in that specific order.

If CSS classes or IDs are directly next to eachother, that means both are required for the rule to be enacted. I am not sure if normal html elements work the same, but I assume not.

Margin/Padding

For Margin or Padding property in CSS, the order is Top/Bottom Right/Left, or Top, Right, Bottom, Left with no commas. Only spaces separate these values.

Clear: Both

Clear both will ignore floats form both sides and place itself below those elements.

Clearfix

The following class may be applied to make the size of the parent element the same as the children

clearfix::after{
clear:both;
content: "";
display:table;}

There are some apparent stipulations that I have not looked into that will cause scrollbars to appear if improper margin and padding settings are employed.

Box-shadow

Box-shadow properties operate as the following:

x-offset:

and

y-offset:

will adjust shadow in x and y directions. Other options include blur, color, and spread. Color changes color, blur increases blur, and spread increases size.

Media Queries

While media queries commonly check for screen size or resolution, they can also check for other things such as orientation (landscape or portrait), or if the document is being printed.

Initial value

If the initial value is used for a rule, it will set the value to the previous, initial value. If there is no initial value that was set by the user, the initial value of the document will be set. This may be useful for font sizes but I'm not sure.

For some reason my settings keep resetting and I need to input this every single time. It is good practice regardless.

Push

The following are directions to push into a repository
First, initialize in the correct directory.

git init

Then, add the new files. This command adds them all:

git add .

Next, create a commit and add a description:

git commit -m "DESCRIPTION"

Use

git remote add origin REMOTE-REPOSITORY-URL

then

git remote -v

to add your own git url that you are uploading to, then verify it.

Finally, push it to your repository

git push origin master

Some of these steps are not necessary if you already did the steps in this section before.
